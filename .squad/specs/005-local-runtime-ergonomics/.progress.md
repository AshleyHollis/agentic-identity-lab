- 2026-05-14 — **T10 SECURITY REVIEW — Trinity — AMENDED**

  **Scope:** FR-05 through FR-07, BFF browser-runtime contract (`/chat/session`, CORS, `userId` rule), and all three env example files.

  **Findings and dispositions:**

  1. **`/chat/session` response — ACCEPTED.** Response schema (`session_id` + `expires_at`) exposes no tokens and no PII. `session_id` is a server-generated UUID4 not derived from any token claim. `expires_at` is a server-generated timestamp unrelated to user identity. No raw bearer token in response. Design is sound.

  2. **CORS wildcard-with-credentials guard — AMENDED (requirements.md FR-06, design.md CORS section, tasks.md T07).** The original design's CORS snippet showed unconditional middleware registration with a static default of `["http://localhost:3000"]` but contained no runtime guard against `CORS_ALLOWED_ORIGINS=*`. Wildcard origins combined with `allow_credentials=True` is a known CORS misconfiguration. Amended: (a) FR-06 now requires a startup `RuntimeError` if `*` appears in `CORS_ALLOWED_ORIGINS`; (b) design.md now shows a conditional middleware registration block with the wildcard guard; (c) T07 adds `test_cors_wildcard_rejected_at_startup` and `test_cors_disabled_in_strict_mode_without_explicit_origins` tests.

  3. **CORS conditional activation — AMENDED (design.md).** FR-06 required CORS to be "only active in AUTH_MODE=mock or when CORS_ALLOWED_ORIGINS is explicitly set", but the design showed a static default without a mode guard — a misalignment that would cause Neo to implement CORS as always-on. Design.md now specifies the Settings loading rule: default to `["http://localhost:3000"]` only when `AUTH_MODE=mock`; default to `[]` (no middleware) when `AUTH_MODE != mock` and env var is unset.

  4. **`userId` display-only rule specification — ACCEPTED.** FR-07 correctly states the rule. The mandatory code comment is specified in design.md. T06 includes `test_chat_session_userid_is_not_authz`.

  5. **T08 test wording — AMENDED (tasks.md T08).** Original wording said "tampered/different `userId` cookie or header" — `userId` is a token claim (sub/oid), not a cookie or header. Left uncorrected, Neo would write a test checking for a non-existent header. Amended to: "two mock tokens with different `sub`/`oid` claim values but identical scp/aud/iss both succeed."

  6. **Env placeholders — ACCEPTED.** All three env example files use all-zero GUIDs (`00000000-0000-0000-0000-000000000NNN`) and `{tenant_id}` brace tokens for issuer/JWKS URLs. No real tenant IDs, client IDs, secrets, or tokens present. Public-safe.

  **Verdict: AMENDED. T06–T08 may proceed after these amendments are acknowledged by Neo.**

  **ADR-M4-02 (Python stdlib health check) — Accepted.**
  The pattern `python -c "import urllib.request, sys; urllib.request.urlopen('http://localhost:8080/healthz') or sys.exit(1)"` is functionally correct:
  - Success path: `urlopen` returns a truthy response object; short-circuit prevents `sys.exit(1)`; script exits 0.
  - Failure path: `urlopen` raises `urllib.error.HTTPError` or `URLError`; uncaught exception causes Python to exit non-zero; health check fails correctly.
  - The `or sys.exit(1)` clause is benign dead code on both paths but is a harmless and widely-used pattern. No amendment required.
  - Container port `8080` is correct for all three services (`agent-gateway` maps host 8081→container 8080).
  - `start_period: 10s` is appropriate for local Python FastAPI startup.
